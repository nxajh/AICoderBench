## 风险与应对

| 风险 | 应对 |
|------|------|
| API 费用超预期 | 先用最小 prompt 跑一轮评估费用，控制模型数量 |
| 某模型 API 不稳定 | 预留重试机制，失败则标记跳过 |
| 题目设计不当（太难/太易） | Phase 2 结束前先自己跑一遍确认难度 |
| 人工评审主观性 | 建立评分 checklist，降低主观偏差 |
| 文章关注度低 | 先在技术社区小范围试水，收集反馈再推广 |
